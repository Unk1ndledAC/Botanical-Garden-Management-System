### 基于数据库的应用程序设计与开发——植物园管理系统
1. 植物园日常管理需要维护植物分类、园区地块、库存养护及游客参观等信息，需要使用数据库进行更便捷的管理
2. 植物信息管理：维护植物中文名、形态描述、产地、生境、花期、用途等。
3. 园区管理：维护园区名称、地理位置、简介。
4. 库存管理：记录每株（批）植物所在园区、数量、种植日期、健康状态。
5. 用户与权限：
- 管理员：可增删改植物/园区/库存，可修改本人密码。
- 游客：无需账号，填写昵称后即可浏览。
6. 查询与统计：支持按中文名模糊查询；选中植物后可实时查看其在各园区的分布及状态。
7. 图形化界面：便于不甚了解信息技术的用户及管理人员使用

### 使用说明
正确安装MySQL8.0是运行本管理系统的重要条件，由于该系统已经打包为可执行文件（位于项目根目录/dist/main.exe），因此无需安装Python环境及相关库。亦可通过运行build.bat文件（位于项目根目录）重新打包可执行文件，或在项目根目录下使用
```bash
# Python 3.12 & tkinter 8.6 & pymysql 1.4.6 recommended
python -m main 
```
指令从代码运行。
默认数据库连接配置信息存储于项目根目录/botanic/config.py内，包含数据库连接的基本配置，如默认数据库用户名和密码等，使用前需要对其进行自定义配置。
默认数据库数据信息和模式结构等内容存储于项目根目录/schema.sql内，包含各表各模式，及初始录入的植物信息，管理员信息等，使用前同样需要对其进行自定义配置，并事先通过MySQL Workbench或MySQL Command Line导入该sql文件。
